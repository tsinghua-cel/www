Overview

BunnyFinder is a novel framework designed to identify incentive flaws in Ethereum's Proof-of-Stake (PoS) protocol with minimal manual effort. Leveraging failure injection techniques, BunnyFinder uncovers both known and new incentive issues, helping to improve Ethereum's stability and security.

Key features include:

  • An automated workflow for attack strategy generation and execution.
  • Comprehensive analysis of results to identify flaws.
  • Support for fine-tuning attack parameters for maximum impact.

Methodology

BunnyFinder employs three main components:

  • Strategy Generator: Semi-automatically creates a vast pool of attack strategies.
  • Strategy Executor: Automates the execution of attack strategies.
  • State Analyzer: Evaluates execution logs to identify potential incentive flaws.

Inspired by software testing techniques like fuzz testing and chaos engineering, BunnyFinder shifts the focus from implementation bugs to design flaws, specifically targeting incentive vulnerabilities in blockchain protocols.

Key Findings

Finding 1: Reproduction of a known attack

We reproduced the Ex-ante Reorganization Attack successfully.

Finding 2: Three new incentive flaws

  • Selfish Mining Attack: Profitable with only 13.4% stake control.
  • Staircase Attack-II: Exploits penalties for honest validators.
  • Pyrrhic Victory Attack: Situations where honest validators incur greater losses than Byzantine validators.

Finding 3: Two implementation flaws

  • Synchronization Deadlock: Prevents validators from normal operation.
  • Erroneous Log Reporting: Misleading reorganization reports.

Responsible disclosure of these findings led to Ethereum Foundation's acknowledgment and updates to the protocol.
